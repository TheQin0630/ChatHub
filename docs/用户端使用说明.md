# ChatHub 用户端使用说明

## 适用对象

本文面向 ChatHub 桌面客户端使用者和答辩演示人员，说明如何启动客户端、连接服务器、加入频道、收发消息、查看 Workspace，以及如何使用本地 mock server 做演示。

## 启动客户端

如果已经拿到打包文件，进入发布目录：

```text
dist/ChatHub-win64/
```

双击运行：

```text
ChatHub.exe
```

如果从 Qt Creator 运行，请打开 `CMakeLists.txt`，选择 Qt 6.10+ kit，然后构建并运行 `ChatHub` 目标。

## 连接服务器

顶部栏包含三个输入项：

- Server IP：服务器地址，例如 `127.0.0.1`。
- Port：服务器端口，例如 `1883`。
- Nickname：当前客户端昵称，例如 `alice`。

点击 `Connect` / `连接` 后，状态灯变为连接中或已连接。连接成功后，左侧频道输入、右侧 Server 刷新等功能会变为可用。

断开连接时点击 `Disconnect` / `断开`。

## 语言和主题

顶部右侧提供：

- `中文 / English`：切换界面语言，选择会保存到本地 `Settings`。
- `Light / Dark`：切换浅色/深色主题，选择会保存到本地 `Settings`。

## 加入频道

1. 确认已经连接服务器。
2. 在左侧频道输入框输入 topic，例如 `room/general`。
3. 点击圆形加号按钮，客户端会向服务器发送订阅请求。
4. 收到服务端确认后，频道会出现在左侧列表，并自动切换为当前聊天频道。

频道名要求：

- 不能为空。
- UTF-8 编码后不超过 64 字节。
- 不能包含空格、制表符、换行符。

## 发送消息

1. 选择一个已加入频道。
2. 在底部输入框输入消息。
3. 点击纸飞机按钮发送。

消息要求：

- 不能为空。
- 打包后的 payload 不超过协议限制。
- 未连接或未加入频道时不能发送。

消息发送后会先显示为发送中，收到服务器确认后变为已发送状态。

## 高级发送选项

底部 `Options` / `选项` 默认收起，点击后可设置：

- `QoS1 ack`：使用扩展发布帧，等待服务端确认。
- `Retain`：请求服务端保留该 topic 最新消息。

这些选项依赖服务端实现。mock server 支持基础演示。

## 接收消息和频道切换

- 自己发送的消息显示在右侧。
- 其他用户消息显示在左侧，并用昵称首字母生成头像。
- 切换频道时，中间聊天区只显示当前频道消息。
- 非当前频道收到消息时，左侧频道列表显示未读数。

## 退订频道

鼠标悬停频道卡片时会显示 `Leave` / `退出` 按钮。点击后客户端发送退订请求，并从本地频道列表移除该频道。

当前实现说明：退订不会删除服务端 topic，只是取消当前客户端订阅。

## Workspace

右侧 Workspace 分为三个视图：

- Overview：展示当前会话概览、频道数量、消息数量等。
- Server：展示服务端 topic 列表、连接列表、订阅者和连接级规则操作。
- Logs：展示本地客户端日志和部分服务端日志查询结果。

Server 页签中的频道列表来自服务端 `TopicListResponse`，运行时暂存在 QML 的 `serverTopicListModel` 中，不是本地文件。

## 本地演示

启动 mock server：

```powershell
python tests\mock_server.py --host 127.0.0.1 --port 1883
```

启动两个 ChatHub 客户端：

1. 两个客户端连接 `127.0.0.1:1883`。
2. 两个客户端加入同一频道，例如 `room/general`。
3. 客户端 A 发送消息，客户端 B 应能收到。
4. 可测试 QoS1 和 Retain 基础行为。

## 常见问题

### 连接失败

检查服务器 IP、端口、server 是否正在运行、防火墙是否拦截 TCP 连接。

### 发送按钮不可用

通常是未连接服务器、未加入频道，或消息输入框为空。

### 频道加入失败

检查 topic 是否为空、是否包含空白字符、是否超过 64 字节，或服务端是否拒绝订阅。

### 打包版打不开

确认运行的是 `dist/ChatHub-win64/ChatHub.exe`，并且同目录下保留 Qt DLL、`platforms/`、`qml/` 等部署文件。
