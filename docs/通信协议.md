# ChatHub 通信协议

## 总览

ChatHub 当前实现使用 TCP 长连接和二进制帧协议。客户端不在 UI 中拼接协议数据，所有帧打包、拆包、校验和解析都集中在：

```text
src/protocol/ProtocolAdapter.h
src/protocol/ProtocolAdapter.cpp
```

网络层 `NetworkClient` 只负责 TCP 字节流收发和按协议拆帧。

## 字节序

所有多字节整数使用大端序，也就是 network byte order。

## 基础帧格式

每个帧由固定 4 字节头部、topic 字节串和 payload 字节串组成：

```text
+---------+-----------+--------------+-------------+---------------+
| type:1B | topicLen:1B | payloadLen:2B | topic:N bytes | payload:M bytes |
+---------+-----------+--------------+-------------+---------------+
```

字段说明：

- `type`：消息类型，1 字节。
- `topicLen`：topic UTF-8 字节长度，1 字节。
- `payloadLen`：payload 字节长度，2 字节。
- `topic`：UTF-8 编码的 topic。
- `payload`：类型相关的二进制数据。

限制：

- `topicLen <= 64`
- `payloadLen <= 2048`
- topic 不能为空，不能包含空格、制表符、换行符。

## 消息类型

| 值 | 名称 | 方向 | 用途 |
|---:|---|---|---|
| 1 | Subscribe | Client -> Server | 订阅 topic |
| 2 | Unsubscribe | Client -> Server | 取消订阅 topic |
| 3 | Publish | Client -> Server / Server -> Client | 普通发布或普通投递 |
| 4 | SubscribeAck | Server -> Client | 订阅确认 |
| 5 | PublishAck | Server -> Client | 普通发布确认 |
| 6 | LogQuery | Client <-> Server | 旧版日志快照查询/响应 |
| 7 | SubQuery | Client <-> Server | 旧版订阅快照查询/响应 |
| 8 | LogListRequest | Client -> Server | 分页请求服务端日志 |
| 9 | LogListResponse | Server -> Client | 分页返回服务端日志 |
| 10 | TopicListRequest | Client -> Server | 分页请求服务端 topic 列表 |
| 11 | TopicListResponse | Server -> Client | 分页返回服务端 topic 列表 |
| 12 | PublishEx | Client -> Server | 扩展发布，支持 QoS1、Retain、packetId |
| 13 | PubRec1 | Server -> Client | QoS1 发布确认 |
| 14 | Deliver | Server -> Client | 扩展投递，支持 QoS1、Retain、packetId |
| 15 | DeliverAck | Client -> Server | QoS1 投递确认 |
| 16 | ConnectionListRequest | Client -> Server | 请求连接列表 |
| 17 | ConnectionListResponse | Server -> Client | 返回连接列表 |
| 18 | TopicSubscribersRequest | Client -> Server | 请求 topic 订阅者 |
| 19 | TopicSubscribersResponse | Server -> Client | 返回 topic 订阅者 |
| 20 | RuleSetRequest | Client -> Server | 设置连接级规则 |
| 21 | RuleSetResponse | Server -> Client | 返回规则设置结果 |
| 22 | SubscribeReject | Server -> Client | 拒绝订阅 |
| 23 | PublishReject | Server -> Client | 拒绝发布 |
| 24 | TopicCreateRequest | Client -> Server | 创建 topic |
| 25 | TopicCreateResponse | Server -> Client | 创建 topic 结果 |
| 26 | TopicDeleteRequest | Client -> Server | 删除 topic |
| 27 | TopicDeleteResponse | Server -> Client | 删除 topic 结果 |
| 28 | FdTopicRelationRequest | Client -> Server | 请求连接和 topic 的关系 |
| 29 | FdTopicRelationResponse | Server -> Client | 返回连接和 topic 的关系 |

## 聊天消息 payload

普通聊天 payload 使用 UTF-8 文本：

```text
nickname<TAB>message
```

解析规则：

- 如果存在第一个 tab，tab 左侧为昵称，右侧为消息内容。
- 如果不存在 tab，昵称按 `remote` 处理，整个 payload 作为消息内容。

## PublishEx / Deliver payload

扩展发布和扩展投递的 payload 内部还有 5 字节扩展头：

```text
+----------+-------------+-------------+----------------+
| flags:1B | packetId:2B | innerLen:2B | inner payload   |
+----------+-------------+-------------+----------------+
```

`flags` 位定义：

- bit 0：Retain。
- bit 1-2：QoS，目前客户端只允许 0 或 1。
- bit 3：Dup。

规则：

- QoS1 时 `packetId` 必须大于 0。
- `innerLen` 不能超过剩余 payload 长度。
- `inner payload` 对聊天消息仍使用 `nickname<TAB>message`。

## PubRec1 / DeliverAck payload

确认帧 payload 长度为 3 字节：

```text
+-------------+----------+
| packetId:2B | code:1B  |
+-------------+----------+
```

当前客户端只读取 `packetId`。

## 分页请求 payload

`LogListRequest` 和 `TopicListRequest` 使用相同的 7 字节请求 payload：

```text
+--------------+----------+----------+---------+
| requestId:2B | offset:2B | limit:2B | order:1B |
+--------------+----------+----------+---------+
```

含义：

- `requestId`：客户端生成的请求编号。
- `offset`：分页偏移。
- `limit`：分页条数，客户端限制为 1 到 50。
- `order`：日志查询中 `1` 表示 newest first；topic 查询中当前用作 include empty 标志。

## 分页响应 payload

`LogListResponse` 和 `TopicListResponse` 共享 9 字节响应头：

```text
+--------------+-------------+----------+-----------------+------------+
| requestId:2B | totalItems:2B | offset:2B | returnedCount:2B | hasMore:1B |
+--------------+-------------+----------+-----------------+------------+
```

### LogListResponse 行格式

每行：

```text
+--------+----------------+
| len:1B | UTF-8 log text |
+--------+----------------+
```

客户端按 `returnedCount` 读取，最多读取 50 行。

### TopicListResponse 行格式

每行：

```text
+------------+----------------+--------------------+
| topicLen:1B | topic:N bytes | subscriberCount:2B |
+------------+----------------+--------------------+
```

客户端将其转换为 QML 可显示的 `{ topic, subscribers }`。

## Reject payload

`SubscribeReject` 和 `PublishReject` payload：

```text
+------------+----------------------+
| code:1B    | reason UTF-8 text    |
+------------+----------------------+
```

当前客户端约定：

- `1`：denied。
- `2`：topic not found。
- 其他：按服务端 reason 展示。

订阅被拒绝且 code 为 2 时，客户端会尝试发送 `TopicCreateRequest` 创建 topic，创建成功或已存在后再次订阅。

## 连接列表 payload

`ConnectionListResponse`：

```text
+----------+------------------------------------+
| count:2B | entries[count]                     |
+----------+------------------------------------+
```

每个 entry：

```text
+-------+-----------+---------+
| fd:4B | ipv4:4B   | port:2B |
+-------+-----------+---------+
```

## Topic 订阅者 payload

`TopicSubscribersResponse` 支持两种格式。新格式：

```text
+-----------+----------+------------------------------------+
| status:1B | count:2B | entries[count]                     |
+-----------+----------+------------------------------------+
```

旧格式：

```text
+----------+------------------------------------+
| count:2B | entries[count]                     |
+----------+------------------------------------+
```

entry 格式与连接列表一致。

`status` 当前识别：

- `0`：OK。
- `1`：topic not found。

## 规则设置 payload

`RuleSetRequest` payload：

```text
+-------+---------+-------+
| op:1B | mask:1B | fd:4B |
+-------+---------+-------+
```

当前客户端使用：

- `op = 1`：添加规则。
- `op = 2`：删除规则。
- `mask` 允许位：`0x02`、`0x04`、`0x08`。

`RuleSetResponse` payload：

```text
+-----------+
| status:1B |
+-----------+
```

`status = 0` 表示成功，其他值表示失败。

## fd-topic 关系 payload

`FdTopicRelationRequest` payload：

```text
+-------+
| fd:4B |
+-------+
```

`FdTopicRelationResponse` payload：

```text
+-----------+-------+---------+
| status:1B | fd:4B | mask:1B |
+-----------+-------+---------+
```

客户端显示 mask 中的关系位：

- `0x01`：subscribed。
- `0x02`：deny sub。
- `0x04`：deny recv。
- `0x08`：deny publish。

## Topic 创建和删除

`TopicCreateRequest`、`TopicDeleteRequest` 不需要额外 payload，topic 放在基础帧 topic 字段中。

响应 payload：

```text
+-----------+
| status:1B |
+-----------+
```

创建 topic 时：

- `0`：创建成功。
- `1`：topic 已存在。
- 其他：失败。

删除 topic 时：

- `0`：删除成功。
- 其他：topic 不存在或失败。

## Mock Server 覆盖范围

`tests/mock_server.py` 支持基础演示协议：订阅、退订、普通发布、扩展发布、QoS1、Retain、旧版日志查询、分页日志查询、分页 topic 查询。

它不完整模拟连接规则、topic 创建/删除、订阅者列表等高级管理接口。
