# 联调与演示指南

本项目的测试、mock 启动方式和答辩演示步骤统一以本文为准。测试不依赖第三方 Python 包。

## 1. 本地 Mock 服务端

Mock 用于本机演示和客户端开发验证，覆盖当前客户端使用的完整协议：频道创建/删除、订阅/退订、普通发布、QoS1、retain、别名、本人连接信息、在线连接和订阅者查询、规则、fd-topic 关系、按 fd 强制退订与通知、日志和频道分页查询。

在 `ChatHub` 目录执行：

```powershell
python .\tests\mock_server.py --host 127.0.0.1 --port 1883
```

随后启动一个或多个 ChatHub，在顶部连接栏填写 `127.0.0.1` 和 `1883`。

停止 mock：在运行 mock 的终端按 `Ctrl+C`。

## 2. 黑盒协议联调

`tests/final_server_integration.py` 是客户端视角的黑盒测试脚本：只建立 TCP 连接、发送协议帧并验证响应，不写入或修改服务端源码。它既能测本地 mock，也能测课程联调服务器。

先启动 mock 后，执行全部协议模块：

```powershell
python .\tests\final_server_integration.py --host 127.0.0.1 --port 1883
```

指定远程服务器：

```powershell
python .\tests\final_server_integration.py --host 服务器IP --port 1883
```

单模块运行：

```powershell
python .\tests\final_server_integration.py --host 127.0.0.1 --port 1883 --module management
```

| 模块 | 验证内容 | 预期输出 |
| --- | --- | --- |
| `identity` | 设置别名、查询服务端确认的 fd/IP/端口/别名 | `[PASS] identity` |
| `publish` | 创建频道、双客户端订阅、普通发布与别名转发 | `[PASS] publish` |
| `qos_retain` | QoS1 投递、ACK、retain 和别名快照 | `[PASS] qos_retain` |
| `management` | 连接/订阅者别名、强制退订、目标通知 | `[PASS] management` |

脚本使用随机 `test/...` 频道。失败时会输出 `[FAIL]` 并返回非零退出码。

## 3. Qt 单元测试

先构建 Debug 目标：

```powershell
cmake --build .\build\Desktop_Qt_6_11_1_MinGW_64_bit_Debug --target ChatHub protocol_tests network_client_tests controller_tests -j 4
```

再在构建目录运行：

```powershell
ctest --output-on-failure
```

| 测试 | 覆盖范围 |
| --- | --- |
| `protocol_tests` | 帧编解码、分页、QoS/retain、别名、连接/订阅者/关系响应解析 |
| `network_client_tests` | TCP 分帧与高频帧读取期间事件循环响应 |
| `controller_tests` | 连接、订阅、发布 ACK/拒绝、断线清理和用户提示 |

## 4. 人工界面演示

1. 启动 mock，打开两个 ChatHub，用不同昵称连接。
2. 两端加入同一频道；互发消息，确认气泡显示发送方别名且频道不串消息。
3. 打开右侧“概览”的“我的连接身份”，确认服务端返回 fd、IP、端口和别名。
4. 修改一端别名并保存；收到确认提示后再发消息，另一端显示新别名。
5. 发送“可靠 + 保留”消息；第三个客户端加入频道，确认收到 retain 消息及原始别名。
6. 在“连接规则”刷新连接，选择 fd 和频道，先查看订阅者，再执行“强制取消该 fd 的订阅”；目标端应收到被移除提示。

## 5. 常见问题

- **连接被拒绝**：确认 mock 终端仍在运行，IP/端口为 `127.0.0.1:1883`。
- **测试端口被占用**：换端口，例如 mock 使用 `--port 18830`，测试和客户端也改为同一端口。
- **别名保存失败**：别名不能超过 32 个 UTF-8 字节；中文通常每字占 3 字节。
- **强制退订没有效果**：先刷新连接和订阅者列表，核对目标 fd、频道和别名。
- **远程联调留下测试数据**：黑盒脚本会创建随机测试频道和 retain 消息；建议在专用联调实例上运行。

## 6. 使用边界

- 别名是展示信息，可重名、可修改、断线失效，不能当作认证身份。
- 强制退订与规则接口没有管理员鉴权，仅限受控教学演示使用。
- Mock 服务端用于客户端联调，不是第三组最终服务端的替代品；对最终服务端联调时应运行第 2 节脚本。
